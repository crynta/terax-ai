# Product

## Register

product

## Users

Developers who want a fast terminal-first workspace with integrated AI, editor, source control, preview, and local or BYOK model support. They work inside terminal-heavy coding sessions, often across local shells, WSL, project files, git state, and agent conversations.

## Product Purpose

Terax is a lightweight AI-native terminal and agentic development environment. It exists to keep terminal work, code editing, source control, local previews, and AI agents in one compact desktop app without telemetry, account requirements, or heavy IDE scope. Success means users can stay in flow, trust the tool surface, and keep resource overhead low.

## Brand Personality

Fast, technical, restrained. The interface should feel confident and utilitarian, with AI treated as a native primitive rather than a decorative sidebar.

## Anti-references

Terax should not feel like a full IDE replacement, a web browser, a generic productivity workspace, or an account-driven cloud product. Avoid heavy IDE chrome, marketing-style dashboard decoration, gratuitous AI spectacle, and UI patterns that pull users away from terminal-first work.

## Design Principles

- Keep terminal work primary: surrounding tools should support the active shell and coding task, not compete with it.
- Earn every dependency and visual layer: lightweight always applies to the interface as much as the binary.
- Make AI feel native: agents, diffs, autocomplete, tools, and project memory should behave like part of the workspace.
- Preserve trust by default: permission, key storage, local model, and no-telemetry choices should be visible when they affect the user.
- Optimize for keyboard-first flow while keeping mouse affordances discoverable.

## Accessibility & Inclusion

Use WCAG AA as the baseline for contrast, focus visibility, labels, and keyboard access. Reduced motion should be respected for nonessential transitions. Because this is a terminal-first product, keyboard navigation, readable low-glare themes, and screen-reader labels for icon-heavy controls are core requirements, not optional polish.
